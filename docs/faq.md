# Preguntas Frecuentes (FAQ)

## General

### ¿Qué es Facthor VikingOZ?

Facthor VikingOZ es un proyecto en desarrollo. Más detalles serán proporcionados cuando el código fuente sea publicado.

### ¿Por qué este es un repositorio temporal?

Este repositorio sirve principalmente para:
- Gestión de issues durante el desarrollo
- Publicar actualizaciones oficiales
- Compartir el código fuente al finalizar
- Proveer documentación y wiki

### ¿Cuándo estará disponible el código fuente?

El código fuente será compartido al finalizar la fase de desarrollo actual. Mantente atento a las actualizaciones oficiales.

## Issues y Contribuciones

### ¿Cómo puedo reportar un problema?

1. Ve a la pestaña "Issues" del repositorio
2. Haz clic en "New Issue"
3. Completa toda la información solicitada
4. Envía el issue

### ¿Puedo contribuir código ahora?

Actualmente estamos en fase de configuración. Las contribuciones de código serán bienvenidas una vez que el proyecto esté en desarrollo activo.

### ¿Cómo puedo sugerir una mejora?

Crea un issue con el label "enhancement" describiendo tu sugerencia en detalle.

## Documentación

### ¿Dónde puedo encontrar la documentación completa?

La documentación está disponible en:
- Este sitio de GitHub Pages
- El directorio `docs/` del repositorio
- La wiki del proyecto (próximamente)

### ¿La documentación está completa?

La documentación está en desarrollo y será actualizada regularmente. Agradecemos sugerencias para mejorarla.

### ¿Puedo contribuir a la documentación?

Sí, las sugerencias para mejorar la documentación son bienvenidas. Crea un issue con tus propuestas.

## Actualizaciones

### ¿Cómo me entero de las actualizaciones?

- Sigue este repositorio en GitHub ("Watch")
- Revisa regularmente la sección de Issues
- Consulta la documentación actualizada

### ¿Con qué frecuencia se actualiza el repositorio?

Las actualizaciones se publicarán según sea necesario durante el desarrollo del proyecto.

## Contacto y Soporte

### ¿Cómo puedo obtener ayuda?

1. Revisa esta FAQ
2. Busca en los issues existentes
3. Crea un nuevo issue si no encuentras respuesta

### ¿Hay un canal de comunicación?

Actualmente, la comunicación se gestiona a través de los issues de GitHub. Otros canales pueden ser añadidos en el futuro.

## Licencia y Uso

### ¿Bajo qué licencia está el proyecto?

La información sobre la licencia será proporcionada cuando el código fuente sea publicado.

### ¿Puedo usar este proyecto?

Los términos de uso serán especificados cuando el proyecto sea oficialmente lanzado.
